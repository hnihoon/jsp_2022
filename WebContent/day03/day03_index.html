1.post_test.html
post_test.html은 사용자가 입력하는 화면의 url
request_test.jsp은 사용자가 입력을 전달받아 처리하는 url
post 방식으로 요청을 보냅니다.
2.get_test.jsp
get_test.html은 사용자의 목목을 출력하는 화면 & 고객번호로 조회할 수 있는 요청 링크 & form을 이용한 조회
get 방식으로 요청을 보냅니다. (데이터의 조회를 위한 처리)
get 과 post 방식의 주요한 또하나의 차이는 보낼수 있는 데이터의 크기 제한이 다릅니다.
3.get_result.jsp
get_test.jsp에서 받은 데이터 출력 및 데이터 오류 시 처리
4.member_update.jsp
1)get_test.jsp -> get_result.jsp에서 내용(전화번호,주소,도시코드)을 수정하고
저장버튼 클릭시 내용이 저장되어 member_update로 이동하게 만들었다. 2)다시쓰기 버튼 클릭시 내용이 처음 내용으로 초기화된다.
3)HrdMemberDao.java에 update메소드 추가해서 전화번호,주소,도시코드 변경문 작성
